## 2026-06-03 - Avoid degrading readability for tiny math micro-optimizations and don't break kinematics

**Learning:** Replacing standard Python math functions like `abs()` with inline conditionals (e.g., `alpha if alpha >= 0.0 else -alpha`) in heavily executed loops (like `equations_of_motion` calculating `CD`) yields unmeasurably small macro-performance gains (~0.005s over 1000 macro iterations) while significantly degrading code readability and violating persona guidelines. Furthermore, in the `Linearizer.compute_jacobian`, it is mathematically incorrect to skip the heading state (`psi`, index 8) perturbation. Although aerodynamic forces are independent of heading on a flat earth, the translational kinematic equations (position rates) are fundamentally dependent on `psi`.

**Action:** Stop hunting for micro-optimizations that offer no real-world measurable impact if the codebase is already heavily optimized. Do not assume any state derivative can be skipped in finite difference Jacobians unless perfectly verified mathematically (like `x` and `y` positions, which truly have no effect on derivatives in a uniform atmosphere). When no suitable optimizations exist, stop cleanly without creating a Pull Request.
