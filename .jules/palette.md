## 2024-03-11 - Unit Toggles and HTML5 Validation Constraints
**Learning:** When dynamically changing input units (e.g., metric to imperial) via toggles, updating the value is not enough. The HTML5 validation attributes (`min`, `max`) and any associated helper text must also be updated simultaneously. Otherwise, the browser's native `:invalid` pseudoclass will incorrectly flag valid converted values as errors because they break the hardcoded default limits (e.g., a max of 1000m/s will incorrectly fail 1943 kts).
**Action:** Always bind HTML5 form validation attributes (`min`, `max`, `step`) and related visual helper text to the same state/event listener that manages unit conversions or data formatting.

## 2024-03-12 - Custom Toggle Switch Label Interactions
**Learning:** Wrapping a complex custom toggle switch (a checkbox acting as a sliding switch between two explicit states like Metric/Imperial) inside a single `<label>` tag creates a poor UX for users attempting to click the adjacent descriptive text. If a user clicks the "Metric" text while it is already on Metric, the browser's default `<label>` behavior blindly toggles the underlying checkbox to Imperial, which is the opposite of the user's intent.
**Action:** For side-by-side binary options representing a switch, decouple the descriptive text from the native `<label>`. Instead, style the text strings as interactive (e.g. `cursor: pointer`, `tabindex="0"`) and attach explicit JavaScript event listeners that deterministically set the intended state (`true` or `false`) regardless of current state.
